# 5. 討論與結論 (Discussion and Conclusion)

本章針對第 4 節實驗結果進行討論，詮釋發現的意涵、與研究問題的對應，說明需改進之處與未來工作，並總結全文。

---

## 5.1 結果詮釋與研究問題對應 (Interpretation and RQ Mapping)

**RQ1（架構與 inner_dim）**：Group Arch 實驗證實 p3_cycles 隨 inner_dim 由 1024 增至 4096 呈合理遞增（2,398→2,650），Path 3 閘級模擬能正確反映架構規模差異；dp 9 在 p2_area 上達最低 (217k µm²)，顯示架構參數對 PPA 的顯著影響。

**RQ2（EDA 策略）**：Group EDA 顯示 p2_dynamic_power 在 176.5–190.0 mW 間變化，dp 5（high effort + retime + clock_gating 等）達最低，證實細粒度 EDA 旗標可有效推動 Pareto 邊界。

**RQ3/RQ4（頻率與時序）**：Group Freq 顯示 p2_timing_slack 隨頻率上升而下降，dp 15（300 MHz）達 Gate 2 邊界仍通過，驗證本架構在 TSMC 40nm ULP 下的高頻極限。

---

## 5.2 意涵與實務啟示 (Implications and Practical Insights)

**Path 3 功耗預測的設計意圖**：框架設計上，Path 3 閘級模擬可依實際波形 toggle 行為進行 PrimeTime PX 功耗分析，以捕捉真實電路切換活動。惟目前實驗因授權限制，尚未執行 PtPX，功耗指標暫以 Path 2 動態功耗替代（見 5.3.4）。

**可修改流程的實用價值**：優化器、評估管道與門檻均可依需求替換，使框架能適應不同約束與目標；黑盒子快速合成與 Gate 1/2 提早停止機制可顯著降低無效評估次數，具實務可操作性。

---

## 5.3 需改進之處與限制 (Limitations and Future Improvements)

本小節著重說明目前框架與實驗的**需改進之處**，作為未來工作方向。

### 5.3.1 貝葉斯最佳化與多層次管道的整合設計

本研究採用固定參數配置進行系統性探索，**尚未設計詳細的貝葉斯最佳化 (BO) 整合方案**。未來需解決以下設計問題：(1) **參數與 Path 的對應**：軟體參數（如 inner_dim、out_channels）、硬體參數（reram_size、frequency）與 EDA 旗標如何映射至 BO 搜尋空間，並處理離散與連續參數的混合；(2) **Path 觸發策略**：何時僅執行 Path 1、何時進入 Path 2、何時啟用 Path 3，需設計明確的決策邏輯（如基於 Gate 1/2 門檻、評估預算或 Pareto 候選篩選）；(3) **多保真度建模**：Path 1（快速）與 Path 2/3（昂貴）的評估成本差異顯著，BO 需納入 multi-fidelity 或 early stopping 機制，以在有限預算下最大化 Pareto 覆蓋率。上述設計均為後續工作重點。

### 5.3.2 準確率訓練不足與 Pareto 前緣的參考價值

**因時間限制，本實驗未對 accuracy 進行多 iteration 的完整訓練**。Gate 1 門檻依賴 Path 1 軟體模擬的準確率，在未充分訓練的模型下，Gate 1 的篩選效果可能失真；且各設計點的 accuracy（0.83–0.94）差異未必反映真實的演算法表現。在此前提下，**目前 Pareto 前緣的比較與排序參考價值有限**，主要用於驗證框架流程與多層次評估的可行性。未來應納入完整訓練流程，使 Pareto 候選具備實務上的設計決策參考價值。

### 5.3.3 硬體參數彈性與搜尋空間限制

**現有硬體 RTL 撰寫時並非以各參數皆可彈性調整為目標**，部分架構參數在設計階段即被固定，導致 DSE 搜尋空間受到相當限制。例如 encoder 維度、記憶體介面寬度、PE 佈局等可能無法隨搜尋參數動態擴展。未來需**修改硬體架構與 RTL**，使關鍵參數可透過巨集或參數化模組動態配置，以支援更寬的搜尋空間，充分發揮全端協同設計的探索潛力。

### 5.3.4 功耗分析與 PtPX 授權限制

**目前因無可用的 PrimeTime PX (PtPX) 授權，未執行基於閘級波形的實際功耗分析實驗**。框架設計上 Path 3 可依 VCS 產生的波形進行 PtPX 動態功耗分析，惟實驗中**暫以 Path 2 邏輯合成報告的 dynamic power 替代**。Path 2 的 report_power 為合成後靜態估算，未反映實際 toggle 行為，與 PtPX 的動態功耗分析存在偏差。取得 PtPX 授權後，應重新執行 Path 3 功耗分析，以取得具閘級保真度的能耗數據。

### 5.3.5 測試平台與週期數精確度

**目前 Path 3 的 testbench 並未完整將 Path 1 軟體訓練所得的 weight 餵入閘級模擬**，而是採用 LFSR 自產生刺激或簡化資料流。在此設計下，**p3_execution_cycles 並非完全精準的推論週期數**，與實際 end-to-end 推論 workload 的週期可能存在偏差。現有週期數主要提供**演算法與架構演進的相對參考**，用於比較不同 inner_dim、不同架構配置下的執行差異，而非作為絕對的矽前驗證數據。未來應建立 Path 1 與 Path 3 的完整資料流對接，使 testbench 能載入真實權重與測試向量，以取得精確的週期與功耗預測。

---

## 5.4 結論與未來工作 (Conclusion and Future Work)

本研究針對 HDnn-PIM 架構提出並實作**可修改的自動化全端協同設計框架**，建立「Path 1 軟體模擬 → Path 2 邏輯合成 → Path 3 閘級模擬」三階段評估管道，並將 EDA 細粒度合成旗標納入搜尋空間。**核心設計邏輯為可修改性**：優化器、評估管道、門檻與資料來源均可依需求替換。實驗以 15 點固定參數配置驗證框架可行性，黑盒子快速合成與 Gate 1/2 提早停止機制具實務可操作性。惟因時間與授權限制，目前存在五項待改進之處（見 5.3）。在此前提下，本框架主要驗證**全端協同設計流程與多層次評估管道的可行性**，為後續完整 DSE 與 Pareto 優化奠定基礎。

未來工作方向如下：

| 方向 | 內容 |
| :--- | :--- |
| **BO 與多層次管道整合** | 設計參數與 Path 的對應、Path 1/2/3 觸發策略與決策邏輯、以及 multi-fidelity 或 early stopping 機制，使 BO 能在有限評估預算下最大化 Pareto 覆蓋率 |
| **完整訓練與 Pareto 驗證** | 納入多 iteration 的 accuracy 訓練，使 Gate 1 篩選具實效、Pareto 前緣具設計決策參考價值 |
| **硬體參數化與搜尋空間擴展** | 修改 RTL 架構，使 encoder 維度、記憶體介面、PE 佈局等可透過巨集或參數化模組動態配置，支援更寬的 DSE 搜尋空間 |
| **PtPX 功耗分析** | 取得 PrimeTime PX 授權後，執行 Path 3 基於閘級波形的動態功耗分析，取代 Path 2 report_power 的靜態估算 |
| **Path 1–Path 3 資料流對接** | 建立 testbench 與 Path 1 軟體訓練的完整對接，使閘級模擬能載入真實權重與測試向量，取得精確的週期與功耗預測 |

此外，可進一步探索實體設計層擴展（Floorplan、Placement、Routing）、架構泛化（擴展至 DNN、GNN 等其他加速器）等方向，以提升框架的完整性與適用範圍。
