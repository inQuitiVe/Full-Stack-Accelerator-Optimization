# 6. 結論與未來展望 (Conclusion and Future Work)

## 6.1 結論 (Conclusion)

本研究針對超維度記憶體內運算 (HDnn-PIM) 架構，提出並實作一套**可修改的自動化全端協同設計框架**。**核心貢獻在於建立可修改的流程**：優化器、評估管道與資料來源均可依需求替換。我們成功克服傳統硬體設計空間探索 (DSE) 面臨的三項挑戰：軟硬體脫節、合成評估過度耗時，以及忽略 EDA 綜合策略的潛力。

**表 6-1：本研究主要貢獻與對應結論**

| 貢獻 | 結論 |
| :--- | :--- |
| 可修改的多維度協同流程 | 打破軟體演算法與硬體 RTL 界線，並首次將 EDA 合成旗標納入探索空間；流程支援修改資料與目標，使探索能發掘單一領域優化無法觸及的 Pareto 最佳解 |
| 可修改的多層次保真度與提早停止 | 以軟體模擬準確率作為 Gatekeeping，有效過濾不具競爭力的設計；門檻與階段可依約束修改，避免後續硬體資源浪費；Path 3 依實際波形 toggle 行為預測功耗，準確度遠優於軟體模擬與合成靜態分析；評估中並採用黑盒子快速合成、EDA 細粒度旗標探索等技巧 |

綜上，本框架在「高維度且昂貴評估」的硬體 DSE 問題上，展現出卓越的搜尋效率與 Pareto 前緣品質；其可修改性使流程能適應不同資料、約束與優化目標。

## 6.2 研究限制 (Limitations)

1. **評估止步於邏輯合成**：目前未納入 Floorplan、Placement 與 Routing，時序與擁塞數據為合成階段估算。
2. **架構專一性**：框架針對 HDnn-PIM 量身打造，擴展至其他加速器架構需額外適配。
3. **保真度層級**：目前為兩層（軟體模擬 + 邏輯合成），尚未實作 Multi-Fidelity BO 的主動採樣策略。

## 6.3 未來工作 (Future Work)

**表 6-2：未來研究方向**

| 方向 | 內容 |
| :--- | :--- |
| **實體設計層擴展** | 將 Floorplan、Placement、Routing 等後端參數納入搜尋空間，取得更精確的時序與擁塞數據 |
| **Multi-Fidelity BO** | 引進 BOCA 或 MF-MES 等演算法，讓 BO 主動選擇在 Fast Mode 或 Slow Mode 下評估，進一步極大化搜尋效率 |
| **架構泛化** | 透過擴充模板機制與可修改流程，使框架適用於 Systolic Arrays、Transformer 加速器等其他類型深度學習硬體 |
